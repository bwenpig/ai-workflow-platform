## 产品定位

### 目标用户
- **AI 应用开发者**：需要快速编排多模型链路的技术人员
- **算法工程师**：需要灵活调整模型参数和数据处理流程
- **技术 Leader/产品经理**：需要可视化评估 AI 链路效果的非编码角色

### 核心价值
1. **拖拽式编排**：无需编码，分钟级调整模型链路
2. **多模型统一**：一套工作流对接可灵、万相、Seedance、NanoBanana 等主流模型
3. **灵活扩展**：支持 Python 脚本节点，自定义数据处理逻辑
4. **即时反馈**：实时查看执行结果，快速迭代优化

### 使用场景
| 场景 | 描述 | 典型用户 |
|------|------|----------|
| AI 生图工作流 | 文生图→图生图→高清修复→后处理 | 算法工程师 |
| AI 生视频工作流 | 文生视频→视频增强→格式转换 | AI 应用开发者 |
| 多模型对比 | 同一输入并行调用多个模型，对比输出质量 | 产品经理 |
| 数据预处理 | 图片裁剪/压缩/格式转换→模型输入 | 算法工程师 |
| 批量处理 | 批量输入→自动排队→结果归档 | AI 应用开发者 |

## 与 dag-scheduler 的关系

### 架构定位

```
┌─────────────────────────────────────────────────────────────┐
│                    ai-workflow (业务应用)                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  画布编辑器 (React Flow)                              │   │
│  │  工作流管理 (CRUD/版本/协作)                          │   │
│  │  执行监控 (实时状态/历史记录)                         │   │
│  │  模型适配器 (可灵/万相/Seedance/NanoBanana/...)       │   │
│  │  Python 脚本节点                                       │   │
│  └──────────────────────────────────────────────────────┘   │
│                          ↓ 使用                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │           dag-scheduler (SPI 框架库)                  │   │
│  │  ┌────────────┐  ┌──────────────┐  ┌──────────────┐  │   │
│  │  │NodeExecutor│  │ExecutorRegistry│  │AutoDiscovery │  │   │
│  │  │   SPI 接口  │  │   注册中心    │  │   自动发现   │  │   │
│  │  └────────────┘  └──────────────┘  └──────────────┘  │   │
│  │  ┌────────────┐  ┌──────────────┐                     │   │
│  │  │ REST API   │  │RemoteExecutor│                     │   │
│  │  │   管理     │  │    Proxy     │                     │   │
│  │  └────────────┘  └──────────────┘                     │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 职责划分

| 模块 | dag-scheduler | ai-workflow |
|------|---------------|-------------|
| **执行器 SPI** | ✅ 提供 NodeExecutor 接口 | ✅ 实现具体模型适配器 |
| **执行器注册** | ✅ 提供 ExecutorRegistry | ✅ 注册模型适配器为执行器 |
| **自动发现** | ✅ 提供@NodeComponent 注解 | ✅ 使用注解标记适配器 |
| **DAG 解析** | ❌ 不负责 | ✅ 负责前端画布→DAG 转换 |
| **任务调度** | ❌ 不负责 | ✅ 负责拓扑排序和执行调度 |
| **依赖管理** | ❌ 不负责 | ✅ 负责节点间数据传递 |
| **业务工作流** | ❌ 不负责 | ✅ 负责完整业务工作流实现 |
| **REST API** | ✅ 提供执行器管理 API | ✅ 提供工作流 CRUD/执行 API |

### 依赖关系

```xml
<!-- ai-workflow/pom.xml -->
<dependency>
    <groupId>com.ben</groupId>
    <artifactId>dag-scheduler</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 扩展方式

ai-workflow 通过实现 dag-scheduler 的 SPI 接口扩展模型适配器：

```java
@Component
@NodeComponent(value = "kling", name = "可灵 AI")
public class KlingAdapter implements NodeExecutor {
    @Override
    public String getType() {
        return "kling";
    }
    
    @Override
    public NodeExecutionResult execute(NodeExecutionContext context) {
        // 调用可灵 API
        // ...
    }
}
```

## 验收原则

1. **功能可验收**：每项功能有明确的验收标准，可测试验证
2. **用户需求导向**：基于用户使用场景定义功能，不是技术实现细节
3. **优先级清晰**：P0 必须实现，P1 建议实现，P2 未来规划
4. **独立可交付**：每个优先级阶段都可独立交付可用版本
