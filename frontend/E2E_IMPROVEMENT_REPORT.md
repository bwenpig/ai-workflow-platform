## 🔧 修复要点总结

### 1. 选择器改进

**修复前：**
```typescript
// 使用 getByText 导致歧义
const node = page.getByText('输入节点')
await expect(node).toBeVisible()
```

**修复后：**
```typescript
// 使用 .first() 避免与画布上的节点混淆
const node = page.getByText('输入节点').first()
await expect(node).toBeVisible()

// 或使用 data-testid
const canvas = page.getByTestId('react-flow')
const nodes = page.locator('[data-testid^="node-"]')
```

### 2. 等待机制改进

**修复前：**
```typescript
// 缺少明确等待
await page.goto('/')
await processNode.dragTo(canvas)
```

**修复后：**
```typescript
// 添加明确的等待条件
await page.goto('/')
await expect(page.getByTestId('canvas')).toBeVisible({ timeout: 10000 })
await expect(page.getByText('节点工具箱')).toBeVisible({ timeout: 10000 })
await processNode.dragTo(canvas, { force: true })
```

### 3. 拖拽操作改进

**修复前：**
```typescript
await processNode.dragTo(canvas, {
  targetPosition: { x: 200, y: 200 }
})
```

**修复后：**
```typescript
await processNode.dragTo(canvas, {
  targetPosition: { x: 200, y: 200 },
  force: true  // 解决遮挡问题
})
```

### 4. 语法错误修复

**修复前：**
```typescript
// 错误的语法
await expect(nodes).toHaveCount({ min: 1 }, { timeout: 5000 })
```

**修复后：**
```typescript
// 正确的语法
const count = await nodes.count()
expect(count).toBeGreaterThanOrEqual(1)
```
